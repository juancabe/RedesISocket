Descripción de la práctica

El API De Sockets

Práctica Sockets: El servico Finger

Objetivos

El objetivo de esta práctica es desarrollar una aplicación en red. Con ello se pretende que el alumno conozca el modelo de programación cliente-servidor y realice una aplicación como usuario del nivel de transporte. Todo ello con una aplicación sencilla que lo capacite para desarrollar en el futuro sus propias aplicaciones cliente-servidor.

Entorno de desarrollo

  • Linux Debian (nogal.usal.es)
  • Sockets de Berkeley
  • Lenguaje de programación C

Especificaciones del protocolo

La  aplicación que vamos a desarrollar implementará el servicio "finger". El servicio finger es un protocolo que proporciona información de los usuarios de una máquina, estén o no conectados en el momento de acceder al servicio.

Condiciones de entrega

Práctica opcional

Las primeras líneas de todos los ficheros fuente (.c y .h) contendrán unas líneas de comentarios donde se especifique el nombre del fichero fuente y el nombre completo y DNI de los autores. Ejemplo:

/*

** Fichero: servidor.c
** Autores:
** Antonio Pérez Puerto DNI 70876543M
** Araceli Sánchez Álvarez DNI 06876701A
*/

Es obligatorio crear un fichero Makefile que genere todos los ejecutables y adjuntarlo en la entrega de la práctica.

Es obligatorio adjuntar el scripts de ejecución. Sin él la práctica no será evaluada.

Entregar un informe en formato PDF que incluya los aspectos que se deseen destacar de la implementación de la práctica y las pruebas de funcionamiento realizadas.

Los ficheros fuente, Makefile, script de ejecución y el informe se comprimirán en un fichero llamado sockets.zip.

El incumplimiento de cualquiera de estas indicaciones, así como de cualquiera de los requisitos especificados supondrá un suspenso en esta práctica.
