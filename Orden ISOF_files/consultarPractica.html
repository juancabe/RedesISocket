Orden Lsof

Orden útil para ver qué proceso utiliza qué puerto

La orden: netstat -an --inet Muestra las conexiones activas sin resolver los números de puerto pero no qué proceso es el que tiene la conexión. Para verlo ejecutaremos la orden lsof.

Sintaxis:

lsof -i[N][PROTOCOLO_TRANSPORTE][@NOMBRE_MAQUINA|DIR_IP]:PUERTO.

Donde:

  • N puede ser 4 ó 6 (versión del protocolo IP)
  • PROTOCOLO_TRANSPORTE puede ser TCP ó UDP
  • NOMBRE_MAQUINA es el nombre del equipo
  • DIR_IP es la dirección IP
  • PUERTO es el número de puerto.

Ejemplos de uso:

  • Por ejemplo, si tenemos lanzado un proceso servidor en nuestra máquina que escucha en el puerto 9999 en TCP la orden sería:
    • lsof -i4TCP:9999

COMMAND PID     USER      FD TYPE DEVICE      SIZE/OFF NODE NAME
servidor     12620 usuario   3u IPv4  114483944 0t0          TCP    *:9999 (LISTEN)

  • En caso de querer mostrar todos los procesos que escuchan en el puerto 9999 la orden sería:
    • lsof -i :9999
COMMAND    PID     USER          FD  TYPE DEVICE      SIZE/OFF NODE NAME
servidor       12620 usuario        3u  IPv4 114483944      0t0     TCP     *:9999 (LISTEN)
servidor       12620 usuario        4u  IPv4 114483945      0t0     UDP    *:9999
NOTA: Es posible que se necesiten permisos de root para que funcione. 

De esta forma, si ahora quisiéramos terminar de forma ordenada dichos procesos sería muy sencillo haciendo una llamada a kill PID_DEL_PROCESO.

  • kill -SIGTERM 12620 o bien killall servidor
